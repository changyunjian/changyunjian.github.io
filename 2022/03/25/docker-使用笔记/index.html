docker 使用笔记

docker是一个虚拟环境容器，类似于虚拟机，但是也有区别。使用docker可以将预先设置好的软件封装，环境封闭，无需再担心各种软件版本、环境配置问题，因此会是以后生信分析得未来之一。但是docker本地环境与虚拟环境的交互较为复杂麻烦，批量运行方法不成熟。

镜像拉取及查看

对目的镜像拉取到仓库中
‘’’shell
docker pull changyunjian/phlat
‘’’
这样，本地便有了此镜像的备份。
查看镜像
‘’’shell
docker imgaes
‘’’

镜像实例化

运行镜像便可获得一个镜像实例，可以对此镜像进行个性化修改
‘’’shell
docker run -it –name {your_name} changyunjian/phlat bash

-it 以交互模式实例化，可以对镜像中的文件进行修改，运行程序。
-d 以后台模式运行，可以直接运用程序。如
docker run -d –name {your_name} changyunjian/phlat bash example.sh
镜像中的example程序将被执行
-v 卷的映射，将本地目录映射到镜像中，方便数据的输入输出
–rm 运行结束直接删除实例
‘’’shell
查看正在运行中的实例
‘’’
docker ps -a
‘’’

保存个性化实例

当你完成对实例的修改，并想要保存为自己的镜像。
‘’’
docker commit -m “描述信息” -a ‘作者’ {实例编号} {新镜像名称}
‘’’
这样便可使用此镜像构建新的实例。

复制文件

将文件从实例复制到本地中，例如
‘’’
docker cp changyunjian/phlat：/opt/example.file ./
‘’’
